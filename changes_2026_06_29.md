## 1. Excalidraw Draw Module (`components/diagrams/DiagramEditor.tsx`)

Several enhancements to the Excalidraw-based diagram editor (`@excalidraw/excalidraw@0.18.1`).

### 1a. Snap-to-grid & spacing/alignment indicators

- Added grid + object-snapping appState flags to the `initialData` (both the
  parsed branch and the fallback branch) and persisted them on save.
- Flags used (all verified present in 0.18.1 type defs):
  - `gridSize: GRID` — draws the visual grid dots (already present).
  - `gridModeEnabled` — snaps shapes to the grid while dragging/resizing.
  - `objectsSnapModeEnabled` — pink alignment + equal-gap spacing guides
    relative to other shapes.
- **Final state:** `gridModeEnabled` is **`false`** (see §1d for why), `gridSize`
  and `objectsSnapModeEnabled` remain on. The save block fallback is
  `appState.gridModeEnabled ?? false`.

### 1b. Image import & clipboard paste

- **Restore bug fix:** `handleRestore` previously called `updateScene` with only
  `elements` + `appState`, dropping embedded image `files`. Restored versions
  with pasted/imported images rendered broken. Now re-registers files via
  `excalidrawAPI.addFiles(Object.values(parsed.files))` before swapping the scene.
- **"Import image" button:** added to the Assets panel. Opens a hidden
  `<input type="file" accept="image/*">`, reads the file to a data URL, scales it
  to fit (max 320px), snaps dimensions + position to `GRID`, registers the file
  via `addFiles`, and inserts an `image`-type element via `updateScene` (cascade
  offset matching the existing asset/stencil placement logic).
- New supporting code: `fileInputRef` (`useRef`), `importImage` callback.
- New imports: `useRef` (react), `ImagePlus` (lucide-react).
- Native clipboard paste (Ctrl/Cmd+V), drag-and-drop, and the toolbar image tool
  already worked — only the restore path and an explicit button were missing.
- **Note:** clipboard paste requires a secure context (works on `localhost`/HTTPS,
  not plain-HTTP LAN). This motivated the HTTPS work in §2.

### 1c. Auto-connectors / elbow arrows (fixed edge connection points)

- Added `currentItemArrowType: "elbow"` (typed `as const` to satisfy the
  `"sharp" | "round" | "elbow"` union) to both `initialData` branches and the
  save block (`appState.currentItemArrowType ?? "elbow"`).
- Effect: newly drawn arrows default to **elbow** routing and show **fixed
  connection points (dots)** on shape edges/midpoints when binding — Excalidraw's
  native arrow binding (arrows attach to shapes and reroute when shapes move).
- **Not implemented (not natively supported):** draw.io/tldraw-style hover
  handles that spawn a connector when hovering a shape. Would require a custom
  canvas overlay; deliberately skipped.

### 1d. Click-jump fix (grid snapping disabled by default)

- **Problem:** with `gridModeEnabled: true`, a tiny accidental drag on click
  (common on trackpads, once past Excalidraw's hardcoded 10px `DRAGGING_THRESHOLD`)
  snapped off-grid elements to the nearest grid line — a visible "jump on click".
- **Fix:** set `gridModeEnabled` to `false` in both `initialData` branches and
  changed the save-block fallback to `?? false`. The grid dots stay visible
  (`gridSize`), object snapping stays on, and placement buttons still snap
  programmatically. Users can re-enable grid snapping per-session via Excalidraw's
  own grid toggle.
- There is **no** configurable click-delay or drag-threshold prop in Excalidraw —
  `DRAGGING_THRESHOLD = 10` is an internal hardcoded const.

## 2. HTTPS on localhost

Goal: satisfy the secure-context requirement so clipboard image paste works.

- **`package.json`:** added a `dev:https` script:
  ```
  "dev:https": "next dev --experimental-https --experimental-https-key ./certificates/localhost+2-key.pem --experimental-https-cert ./certificates/localhost+2.pem"
  ```
- **Tooling installed (Homebrew, not npm — does not touch the Nexus registry):**
  `mkcert` + `nss`.
- **Certificates generated** into `certificates/`:
  `localhost+2.pem` and `localhost+2-key.pem` (valid for `localhost`,
  `127.0.0.1`, `::1`). `*.pem` is already gitignored.
- **Manual step left to the user:** `mkcert -install` (needs an interactive
  sudo/keychain password) to trust the local CA and remove the browser warning.
- Background: Next's built-in `--experimental-https` (bundled mkcert) failed at
  the CA-install step, so we switched to a Homebrew mkcert + explicit cert paths.

## 4. "Main Diagram" per Asset (hero architecture diagram)

A single, nullable pointer from each asset to one featured Excalidraw diagram.
Implemented as an FK-style column on `assets` (same pattern as `tier_id` etc.),
distinct from the existing many-to-many `diagram_assets` junction.

### 4a. Schema (`lib/db.ts`)

- Added an idempotent boot migration:
  ```sql
  ALTER TABLE assets
    ADD COLUMN IF NOT EXISTS hero_diagram_id CHAR(36) NULL
    AFTER icon
  ```
- No hard FK constraint (consistent with the codebase); nulled application-side
  on diagram delete.

### 4b. Types (`types/index.ts`)

- `Asset` interface gained `heroDiagramId: string | null` and
  `heroDiagramName: string | null`.

### 4c. Assets API

- **`app/api/assets/route.ts`** (list GET): added
  `LEFT JOIN diagrams hd ON hd.id = a.hero_diagram_id`, selected
  `hd.name AS hero_diagram_name`, and mapped `heroDiagramId`/`heroDiagramName` in
  `rowToAsset`. **POST**: accepts `heroDiagramId`, persists `hero_diagram_id`
  (added column + matching `?` placeholder + value, now 29 columns/values).
- **`app/api/assets/[id]/route.ts`** (single GET): same JOIN/select + asset
  object fields. **PUT**: accepts `heroDiagramId`, updates `hero_diagram_id`, and
  records old/new `heroDiagramId` in the audit payload.

### 4d. Diagram delete safety (`app/api/diagrams/[id]/route.ts`)

- Before deleting a diagram, clears any asset featuring it:
  ```sql
  UPDATE assets SET hero_diagram_id = NULL WHERE hero_diagram_id = ?
  ```
  (mirrors the existing lookup-delete nulling pattern.)

### 4e. Edit modal (`components/assets/AssetModal.tsx`)

- `AssetFormState`, `EMPTY_FORM`, and `assetToForm` gained `heroDiagramId`.
- New **optional** prop `diagrams?: Diagram[]` (defaults to `[]`) so the other
  modal callers compile unchanged.
- Added a **"Main diagram"** `<SelectField>` in the Links & Notes section
  (options = all diagrams, plus a "— None —" entry) with a helper caption.
- Imported the `Diagram` type.

### 4f. Asset registry page (`app/(dashboard)/assets/page.tsx`)

- Added `diagrams` state, fetched `/api/diagrams` in the parallel load, and passed
  `diagrams={diagrams}` to `AssetModal`. Imported the `Diagram` type.

### 4g. Asset detail page (`app/(dashboard)/assets/[id]/page.tsx`)

## 5. Confluence Sync — embed the main architecture diagram

Extends "Sync to Confluence" so an asset's main diagram is published as an image
under the page's **Overview** heading (Description → meta table → **Architecture**
sub-heading → image).

- **Rendering is client-side** (Excalidraw `exportToBlob` needs a canvas; the API
  can't render the JSON). New helper `components/diagrams/exportDiagramPng.ts`
  renders the stored scene to a base64 **PNG** (white background, 2× scale, 16px
  padding); returns null for empty/unparseable scenes.
- **Asset detail page** (`ConfluenceSync` in `app/(dashboard)/assets/[id]/page.tsx`):
  on sync, if the asset has a `heroDiagramId`, it fetches the diagram content,
  renders the PNG, and includes it as `diagramPng` (base64) in the POST. No main
  diagram → nothing extra sent. Diagram fetch/render failure is non-fatal.
- **Sync route** (`app/api/assets/[id]/confluence-sync/route.ts`): reads
  `diagramPng` from the body and passes it as the new 5th arg to
  `pushAssetToConfluence`.
- **`lib/confluence.ts`:**
  - `pushAssetToConfluence(asset, pageId, settings, syncedAtIso, diagramPngBase64?)`
    — after the page GET, uploads the PNG (when provided) as an attachment, then
    PUTs the body referencing it. Upload failure is logged and **non-fatal**.
  - `uploadDiagramAttachment(...)` — multipart upload to
    `/rest/api/content/{pageId}/child/attachment`; looks up an existing attachment
    by filename and POSTs to its `/data` endpoint to add a version (replace),
    otherwise creates it. Uses a dedicated header set (no `Content-Type`, with
    `X-Atlassian-Token: no-check`). Fixed filename `architecture-diagram.png`.
  - `buildAssetStorageBody(asset, syncedAtIso, diagramFilename?)` — when a filename
    is passed, inserts `<h3>Architecture</h3>` + an `<ac:image><ri:attachment …/>`
    macro (centered, 760px) into the Overview block.
- **Docs:** added a **Confluence Sync** section to `CLAUDE.md`, two Implemented
  Features bullets (main-diagram-per-asset; Confluence sync incl. diagram image),
  and a backlog item for the outstanding test coverage (below).
